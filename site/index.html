For agents & builders

Your agent can buy a read in three requests.

Every service is a standard x402 endpoint — unpaid calls return a 402 challenge, signed replays settle on X Layer. No API keys, no accounts. If your agent holds USDT, it's a customer.

# 1 — call it. unpaid requests return a 402 payment challenge
curl -i -X POST https://optic-production-5675.up.railway.app/v1/rug \
  -H "Content-Type: application/json" -d '{"query":"<token>"}'

# 2 — sign the challenge with an OKX agentic wallet
onchainos payment pay --payload "<PAYMENT-REQUIRED header value>"

# 3 — replay with the signature. verdict + card return, payment settles on-chain
curl -X POST …/v1/rug -H "PAYMENT-SIGNATURE: <authorization_header>" \
  -H "Content-Type: application/json" -d '{"query":"<token>"}'
